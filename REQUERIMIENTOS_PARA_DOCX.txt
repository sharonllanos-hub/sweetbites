MÓDULO 1: AUTENTICACIÓN Y USUARIOS
===================================================================

RF-AU-01: Registro de Usuario
Prioridad: Alta | Estado: ✓ Implementado

Descripción:
El sistema permite que nuevos usuarios se registren proporcionando sus datos personales.

Criterios de Aceptación:
  ✓ El usuario puede ingresar: nombre completo, correo electrónico, contraseña y teléfono (opcional)
  ✓ Validación de email único en la base de datos
  ✓ Validación de contraseña segura (mínimo 8 caracteres, mayúscula, minúscula, número, carácter especial)
  ✓ Contraseña encriptada con bcrypt
  ✓ Mensajes de error claros y específicos
  ✓ Asignación automática de rol "usuario"
  ✓ Redirección a página de login tras registro exitoso

Endpoint: POST /api/auth/register

-------------------------------------------------------------------

RF-AU-02: Inicio de Sesión
Prioridad: Alta | Estado: ✓ Implementado

Descripción:
Los usuarios registrados pueden iniciar sesión con sus credenciales.

Criterios de Aceptación:
  ✓ Login con email y contraseña
  ✓ Validación contra base de datos MySQL
  ✓ Generación de token JWT (válido 7 días)
  ✓ Almacenamiento de token en localStorage
  ✓ Redirección según rol (Admin → Dashboard admin, Usuario → Página principal)
  ✓ Mensajes de error descriptivos

Endpoint: POST /api/auth/login

-------------------------------------------------------------------

RF-AU-03: Cerrar Sesión
Prioridad: Media | Estado: ✓ Implementado

Descripción:
Los usuarios pueden cerrar su sesión activa.

Criterios de Aceptación:
  ✓ Limpieza de token del localStorage
  ✓ Limpieza de sessionStorage
  ✓ Redirección a página principal
  ✓ Mensaje de confirmación

Endpoint: Frontend: authService.logout()

-------------------------------------------------------------------

RF-AU-04: Ver Perfil de Usuario
Prioridad: Alta | Estado: ✓ Implementado

Descripción:
Los usuarios pueden ver su información personal y estadísticas.

Criterios de Aceptación:
  ✓ Visualización de datos personales (nombre, email, teléfono, foto, biografía, rol, plan, fecha de registro)
  ✓ Estadísticas del usuario (recetas creadas, favoritos, comentarios)
  ✓ Badge visual según rol (admin/usuario)
  ✓ Badge visual según plan (gratis/premium)

Endpoint: GET /api/users/profile

-------------------------------------------------------------------
